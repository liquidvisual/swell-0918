What Are Sections?

A section represents a component's top-most layer. It's essentially the 'wrapper' for any standard component using the <section> element with a class of .section.


Not every component is a section though

For instance the <header> and <footer> are special layout components and use their own tags respectively. The hero components, despite being sections do not have the .section class applied since their inner workings differ to the standard components which are predictable.

A .section class creates top and bottom padding as well as a hook for additional class modifiers.


ALL sections have access to the following class modifiers:
  • .theme-[x] - any section can have a theme applied
  • .has-image-bg - any section can have a background image applied with the asset referenced with inline styles. Eg. style="background-image: url(/path/to/asset/)"
  • .is-tinted - applies a tint to any background image with a .has-image-bg class
  • .is-fixed - applies fixed background behavior to any background image with a .has-image-bg class

Text Classes

One can leverage Bootstrap Utility classes pretty much anywhere, even outside of sections. The most common will likely be:

  • .text-[left,center,right]

Cards Component

The Cards Component is a basic collection of 2 or 3 cards without any advanced behavior. Content should be kept to short paragraphs and not used to create lengthy 'text columns' since the equal heights will make cards appear too narrow.

Modifiers:
  • .is-flat - this will strip cards of their depth and shadow.
  • 2 or 3 columns - this is drawn with template code. Pop open the include to see how this is achieved. The default is Three.

Text Component

The text component is the most basic component, comprised of a <h2> heading and a preceding .wysiwyg content section. Links and Primary Buttons inside .wysiwyg sections can have their appearanced changed through themes - although it's recommended they remain the initial $primary value set by the user.

To insert other coloured buttons which are not affected by theme, the user can select .btn.btn-secondary, .btn.btn-success etc which can be set in the Sass.

Text Media Component

This component is a .wysiwyg accompanied by an image or video. The user has to specify an image as a thumbnail. They can also specify a URL for YouTube, Vimeo or Google Maps. If this is left blank, it will default to the image's path.
